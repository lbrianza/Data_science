SARIMA is used on univariate time series, so for the moment we will only consider the column "revenue" and drop all the others.

Grid search¶

The SARIMA model includes several parameters:

SARIMA(p,d,q)(P,D,Q)m

(p,d,q) are the trend elements, where

  • p is the trend autoregressive order (the number of time lags)
  • d is the trend difference order (the number of times the data have had past values subtracted)
  • q is the order of the moving-average model

(P,D,Q)m are the seasonal elements, where

  • P is the seasonal autoregressive order
  • D is the seasonal difference order
  • Q is the order of the seasonal moving-average model
  • m is the number of steps for each seasonal period

in our case, m = 4, since we have 4 quarters in one year. The other parameters can be found using a grid search. The metric used in the grid search is the AIC (Akaike Information Criterion) value, which measures how well the model fits the data while taking into account the overall complexity of the model. We will take the combination of parameters with lowest AIC:

def sarima_grid_search(y,seasonal_period,exog=None):
    p = range(0, 3)
    d = range(1, 2)
    q = range(0, 3)
    pdq = list(itertools.product(p, d, q))
    seasonal_pdq = [(x[0], x[1], x[2],seasonal_period) for x in list(itertools.product(p, d, q))]
    
    mini = float('+inf')
    
    
    for param in pdq:
        for param_seasonal in seasonal_pdq:
            try:
                mod = sm.tsa.statespace.SARIMAX(y,exog=exog,
                                                order=param,
                                                seasonal_order=param_seasonal,
                                                enforce_stationarity=True,
                                                enforce_invertibility=True)

                results = mod.fit()
                
                if results.aic < mini:
                    mini = results.aic
                    param_mini = param
                    param_seasonal_mini = param_seasonal

                print('SARIMA{}x{} - AIC:{}'.format(param, param_seasonal, results.aic))
            except:
                continue
    print('The set of parameters with the minimum AIC is: SARIMA{}x{} - AIC:{}'.format(param_mini, param_seasonal_mini, mini))

Prediction¶

def sarima_eva(y,order,seasonal_order,seasonal_period,pred_date,exog=None):
    # fit the model 
    mod = sm.tsa.statespace.SARIMAX(y,exog=exog,
                                order=order,
                                seasonal_order=seasonal_order,
                                enforce_stationarity=True,
                                enforce_invertibility=True)

    results = mod.fit()
    
    # One-step ahead forecast: forecasts at each point are generated using the full history 
    #up to that point.
    pred = results.get_prediction(start=pd.to_datetime(pred_date), dynamic=False)
    pred_ci = pred.conf_int()
    y_forecasted = pred.predicted_mean
    print(f"PREDICTED: \n{y_forecasted}")
    mse = ((y_forecasted - y) ** 2).mean()
    print('Root Mean Squared Error of SARIMA with season_length={} and dynamic = False : {}'.format(seasonal_period,round(np.sqrt(mse), 2)))

    ax = y.plot(label='observed')
    y_forecasted.plot(ax=ax, label='One-step ahead Forecast', alpha=.7, figsize=(14, 7))
    ax.fill_between(pred_ci.index,
                    pred_ci.iloc[:, 0],
                    pred_ci.iloc[:, 1], color='k', alpha=.2)

    ax.set_xlabel('Date')
    ax.set_ylabel('Sessions')
    plt.legend()
    plt.show()
 
    # Dynamic forecast: use information from the time series up to a certain point, 
    # and after that, forecasts are generated using values from previous forecasted time points.
    pred_dynamic = results.get_prediction(start=pd.to_datetime(pred_date), dynamic=True, full_results=True)
    pred_dynamic_ci = pred_dynamic.conf_int()
    y_forecasted_dynamic = pred_dynamic.predicted_mean
    mse_dynamic = ((y_forecasted_dynamic - y) ** 2).mean()
    print('Root Mean Squared Error of SARIMA with season_length={} and dynamic = True : {} :'.format(seasonal_period,round(np.sqrt(mse_dynamic), 2)))

    ax = y.plot(label='observed')
    y_forecasted_dynamic.plot(label='Dynamic Forecast', ax=ax,figsize=(14, 7))
    ax.fill_between(pred_dynamic_ci.index,
                    pred_dynamic_ci.iloc[:, 0],
                    pred_dynamic_ci.iloc[:, 1], color='k', alpha=.2)

    ax.set_xlabel('Date')
    ax.set_ylabel('Sessions')

    plt.legend()
    plt.show()
    
    return (results)

def forecast(model,y,predict_steps,exog=None):
    
    pred_uc = model.get_forecast(steps=predict_steps,exog=exog)

    #SARIMAXResults.conf_int, can change alpha,the default alpha = .05 returns a 95% confidence interval.
    pred_ci = pred_uc.conf_int()

    ax = y.plot(label='observed', figsize=(14, 7))
    pred_uc.predicted_mean.plot(ax=ax, label='Forecast')
    ax.fill_between(pred_ci.index,
                    pred_ci.iloc[:, 0],
                    pred_ci.iloc[:, 1], color='k', alpha=.25)
    ax.set_xlabel('Date')
    ax.set_ylabel(y.name)

    plt.legend()
    plt.show()
    
    # Produce the forcasted tables 
    pm = pred_uc.predicted_mean.reset_index()
    pm.columns = ['Date','Predicted_Mean']
    pci = pred_ci.reset_index()
    pci.columns = ['Date','Lower Bound','Upper Bound']
    final_table = pm.join(pci.set_index('Date'), on='Date')
    
    return (final_table)

Notes¶

These data presents a particular challenge: due to the COVID-19 crisis, the last 2 observed data points (April 2020 and July 2020) deviate significantly from the trend of the previous years (in particular, revenues are much lower). Approach 3: Use RNN (LSTM) for multivariate time series forecast¶

The main drawback here is that we only have about 15 data points, while neural networks usually require much more statistics, so again, these results should be taken with a pinch of salt.

Conclusions¶

At a first glance, the 3 approaches seem to give different results - however, it's quite the opposite. In fact, we need to distinguish between the 2 different situations:

A) Prediction of 31-10-2020 revenue using NO information from 31-10-2020. In this case, the two models used give a similar prediction for the revenue:

  • SARIMA: 1244.0 (C.I: 742-1746)
  • LSTM: 1233

B) Prediction of 31-10-2020 revenue USING information from 31-10-2020. Here again the two models used give a similar prediction:

  • SARIMA(X) + Exogeneous variable: 1536 (CI: 1502-1569)
  • Linear regression: 1559

In case 2, the predicted value for the revenue is higher - this is due to the fact that we observed a partial 'rebound' in the economy in the last period, as we can see in the 'WallSt' variable - so if we use this additional piece of information in our model, we get a more optimistic prediction.

In conclusion: when predicting the revenue value for a certain data point, IF additional info about that data point are known (as in this case), situation B gives better results and should be preferred. If this is not the case, then only situation A can be used.
